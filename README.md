# “墨韵”读书会书籍共享平台

​					 																																													**TopSECase**



>作为本科生导师计划中重要的一个环节，“墨韵”读书会提供了一个良好的读书交流平台。导师和学生往往不能有很多的交流方式，同时学生的读书笔记也难以及时传递到导师手中。学生和导师互相推荐感兴趣的书籍，需要一个基于网络的平台。平台中包含的个人主页、圈子和书库，很好地涵盖了读书会生活的主要部分，能为读书会提供便利的条件。

## 功能核心

### 用户机制

- 用户应该分为导师和学生
- 用户应该隶属于某个圈子
- 导师才能创建圈子
- 后台有管理员

### 发表类型

#### 日志

任意用户都可以发表日志（浏览权限可设置，默认全部可见）

#### 留言

任意用户可以给其他用户留言，仅对方和自己可见

#### 讨论

任意用户可以在圈子内参与讨论，内容仅圈子内用户可见

#### 图书分享

任意用户可发布图书分享贴，分享贴下任意用户可发布书评

#### 站内消息

在角落广播，全员可见

### 重要功能

#### 查询

站内图书、圈子、日志都应该有查询机制

#### 发布内容流水线

上面提到的多种发表内容的渠道，其中共性的流水线必须坚实可靠

#### 权限识别

上面的行为多数有权限之分，为此要设置相应的辨识符

## 数据结构

这是典型的用户——用户架构平台，也就是用户关系主导的平台。在设计数据结构时要注意以用户为核心，规划动作产生的数据。特别注意构思合理的数据结构，使得用户的数据编辑权限可清晰规划。

## 测试重点

以上提到的重点内容和数据结构都是测试重点。
